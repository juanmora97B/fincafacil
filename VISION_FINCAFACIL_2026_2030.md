## 5. Decisiones Críticas (Que Determinan Éxito)

### 5.1 Decisión 1: Vertical Focus (Ganadería) vs Horizontal (Agro General)

**Nuestra posición (RECOMENDADA):** Stay vertical (ganadería)

```
Razones:
├─ Mejor defensibilidad: Ganadería = datos + IA + network
├─ Profundidad > Amplitud: Expertos en mastitis mejor que básico en 10 crops
├─ Economics mejores: Ganadero paga 10x más que agricultor
└─ Network efectos más fuertes: Vets connect ganaderos

Riesgo aceptado:
└─ TAM menos grande (pero suficiente para $40B+)

Alternativa (NOT RECOMMENDED): Horizontal agrícola
├─ TAM más grande (5x)
├─ Pero: competencia brutal (Deere, agrify, etc)
├─ Economics peores (margen de utilidad más bajo)
└─ Death by a thousand papercuts (100 crop types, cada uno diferente)
```

### 5.2 Decisión 2: Proprietary Data vs Data Steward

**Nuestra posición (RECOMENDADA):** Proprietary (datos son nuestros)

```
Modelo Proprietary (Actual):
├─ Customer data propiedad FincaFácil
├─ Usamos para mejorar IA global
├─ Revenue sharing con customers (si quieren)
├─ Moat defensivo enorme
└─ Risk: Regulación futura (gobierno exige open data)

Modelo Data Steward (Alternativa):
├─ Customer data = propiedad customer
├─ FincaFácil es solo intermediario
├─ Customers pueden exportar cuando quieran
├─ Trust alto, pero IA no mejora (siloed data)
└─ Risk: Competitor entrada con better IA (tiene datos mejores)

Nuestra apuesta:
└─ Stay proprietary, pero transparent (show how data improves models)
└─ Compliance stricta (GDPR-style privacy)
└─ Trust vía transparency (public research, external audits)
```

### 5.3 Decisión 3: Build vs Partner para Expansion Regional

**Nuestra posición (RECOMENDADA):** Hybrid (build teams + partner channels)

```
Build (Equity FTE)
├─ Pros: Control, culture consistency, learning
├─ Cons: Slow, expensive, talent constraints
└─ Aplicar a: Engineering, product, core sales

Partner (Channel, Resellers)
├─ Pros: Fast entry, local networks, risk transfer
├─ Cons: Quality control, revenue share, dependency
└─ Aplicar a: Regional support, integrations, marketing

Mixtura recomendada:
├─ Core (HQ Bogotá): 100% build
├─ Regional hubs (México, Buenos Aires): 70% build, 30% partners
├─ Periférico (Chile, Perú): 30% build, 70% partners
└─ Result: Balance velocidad + control
```
